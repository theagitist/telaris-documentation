# Como aparece nos telefones

Uma página Hotglue fica disposta no tamanho em que você a constrói. Quem visita, porém, chega em telas de toda largura, e muita gente em telefones. Este capítulo explica o que acontece com o seu layout numa tela pequena, e como criar pensando nisso.

> [!note] Ver versus editar
> Este capítulo é sobre **ver** uma página pronta numa tela pequena, o que funciona bem. **Editar** uma página é diferente: faça isso num computador ou tablet, como observado em *Entrando*. São duas experiências separadas.

## O que quem visita vê numa tela pequena

Quando uma página publicada é mais larga que a tela que a mostra, o Hotglue encolhe a página inteira para caber na largura da tela, mantendo cada objeto na mesma posição relativa. A sua organização cuidadosa é preservada; ela apenas aparece menor. Esse encolhimento só vai até certo ponto: abaixo de uma escala mínima, o Hotglue para de encolher e deixa a página rolar para os lados, então a página nunca fica pequena demais para ler.

Isso acontece automaticamente e apenas na visão publicada. Nunca altera nada no editor, e nunca modifica o seu layout salvo. Você sempre edita em tamanho inteiro.

## Dicas de criação para telefones

Como a página é dimensionada como uma peça única, em vez de ser reorganizada, alguns hábitos resultam num layout que sobrevive ao encolhimento:

- **Mantenha o texto grande o bastante para ler depois de encolher.** Se um bloco de texto é pequeno numa página larga, ele ficará muito pequeno num telefone. Na dúvida, deixe-o maior.
- **Evite uma página desnecessariamente larga** se o público de telefone importa. Quanto mais larga a página, mais ela precisa encolher para caber, e menor tudo fica. Uma página mais estreita encolhe menos.
- **Agrupe objetos relacionados** para que fiquem legíveis juntos quando dimensionados, em vez de espalhar uma única ideia por toda a largura de uma tela muito larga.
- **Faça uma prévia num telefone.** Abra a página publicada (não o editor) num telefone de verdade, ou estreite a janela do seu navegador, para ver o resultado dimensionado. Use **Ver no navegador** na visão de Conteúdo hotglue.

> [!tip] Largo e alto é ok, só seja deliberado
> O Hotglue lida bem com páginas muito grandes, e uma tela vasta pode ser exatamente o objetivo. Apenas lembre-se de que, quanto maior a página, mais um telefone precisa encolhê-la. Decida o trade-off de propósito, e não por acidente.
