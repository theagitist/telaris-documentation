# Auto-registro de editores

Además de crear cuentas a mano, una instancia puede dejar que las personas **se registren por sí mismas** como editoras. Esto está desactivado por defecto. Cuando lo activas, quien visita puede solicitar una cuenta de edición a través de un formulario, confirmar su correo, y empezar a editar, sin que tú crees la cuenta primero. Este capítulo cubre cuándo usarlo y cada interruptor que lo gobierna.

El auto-registro se configura desde la consola de administración (los ajustes de auto-registro; en la mayoría de las instancias están junto a los demás controles de cuentas). Toda la función es una sola decisión con varios diales: *¿deberían las personas desconocidas poder volverse editoras aquí, y bajo qué límites?*

## Cuándo usarlo

Actívalo para una convocatoria abierta: un archivo comunitario que invita a colaboradoras, un curso donde el estudiantado se registra por su cuenta, un proyecto público que quiere una barrera baja para ayudar. Déjalo desactivado para una instancia cerrada donde cada persona editora deba ser alguien que añadiste deliberadamente. Puedes activarlo por un tiempo (lo que dura un curso, el periodo de una convocatoria) y volver a desactivarlo; las cuentas existentes no se ven afectadas cuando lo haces.

## Activarlo

El control de auto-registro tiene un interruptor maestro. Mientras está apagado, el formulario de registro está cerrado: quien lo alcanza ve un aviso de que el registro no está abierto, y el enlace de registro se oculta de la página de inicio de sesión. Mientras está encendido, el formulario está abierto y el enlace aparece.

Activarlo revela los límites de abajo. Fíjalos antes de anunciar la convocatoria, no después.

## Los límites

- **Cupo.** Un número máximo de editoras auto-registradas. Una vez alcanzado el cupo, el formulario se cierra solo y el enlace de registro desaparece hasta que subes el cupo o eliminas algunas cuentas. Úsalo para que una convocatoria abierta no se te escape de las manos. El cupo cuenta a las editoras auto-registradas, no a las cuentas que creaste tú.
- **Lista de dominios de correo permitidos.** Una lista opcional de dominios de correo que pueden registrarse (por ejemplo, el dominio de una universidad para un curso). Cuando se fija, solo las direcciones en esos dominios pueden completar el registro; a las demás se les rechaza. Déjala vacía para permitir cualquier dirección.
- **Nombre de la galaxia personal.** Cuando alguien se registra, Telaris puede crear una **galaxia personal** para que trabaje en ella. Tú eliges cómo se nombra: a partir del nombre completo de la persona, o solo de su nombre de pila. Solo el nombre de pila es el valor por defecto que preserva la privacidad, porque el nombre de una galaxia es público; un nombre completo en el título de una galaxia es visible para cada persona que visita. Elige nombres completos solo cuando esa exposición sea aceptable para tu contexto.
- **Acceso predeterminado.** A las nuevas editoras auto-registradas se les pueden dar asientos en galaxias de demostración, de solo lectura o de lectura y escritura, para que aterricen con algo que mirar o en lo que trabajar. *Control de acceso de edición* explica los asientos.

## Cómo inicia sesión una editora auto-registrada

Una editora auto-registrada **no tiene contraseña**. Inicia sesión cada vez con el enlace por correo: introduce la dirección, recibe un enlace de un solo uso, lo sigue. Por eso el correo debe estar configurado para que el registro funcione siquiera; si la instancia no puede enviar correo, nadie puede confirmar un registro ni iniciar sesión después. El capítulo *Ajustes globales* cubre el correo.

Como el registro depende por completo del correo, el flujo es: la persona envía el formulario, Telaris le escribe para confirmar que la dirección es suya, y solo después de que sigue esa confirmación es una editora real. Un registro sin confirmar aún no es una cuenta y se limpia automáticamente si nunca se confirma.

## Verificación

Una editora auto-registrada puede editar en cuanto confirma su correo; no tienes que aprobarla primero. La **verificación** es un paso aparte y opcional que dice "reconozco a esta persona". Cuando verificas una cuenta:

- la persona puede fijar una contraseña, así ya no queda limitada a los enlaces por correo, y
- Telaris le avisa de que ha sido verificada.

La verificación **no** condiciona la edición. Una editora auto-registrada sin verificar ya es una editora completa; la verificación es una marca de reconocimiento y una comodidad (la contraseña), no un permiso. Las cuentas sin verificar se resaltan en la lista de cuentas para que veas de un vistazo quién se ha unido pero aún no ha sido reconocida.

## Mantenerlo a salvo

- **Un cupo es tu válvula de seguridad.** Un formulario abierto sin cupo puede inundarse. Fija un cupo con el que te sientas cómoda moderando, y súbelo deliberadamente.
- **La lista de dominios acota la convocatoria.** Para un curso o una organización, una lista de dominios de correo permitidos convierte "cualquiera" en "cualquiera de aquí", que suele ser lo que en realidad querías decir.
- **Nombres de pila por defecto.** Salvo que tengas una razón para lo contrario, mantén el nombre de la galaxia personal en el nombre de pila, para que el nombre completo de nadie se vuelva el título de una galaxia pública sin su consentimiento.
- **Desactívalo cuando la convocatoria termine.** Un registro dejado abierto indefinidamente es una invitación permanente que puede que hayas olvidado. Ciérralo cuando la razón para abrirlo haya pasado; las cuentas ya creadas se quedan.
